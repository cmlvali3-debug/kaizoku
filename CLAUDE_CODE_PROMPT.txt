Tu es en train de travailler sur Oitaku Import, un bot de recherche de voitures japonaises (JDM) et coréennes (KDM) pour un garage d'importation en France.

═══════════════════════════════════════════════════════════
  CONTEXTE DU PROJET
═══════════════════════════════════════════════════════════

Le projet est déjà structuré. Voici ce qui existe :

FRONTEND (frontend/)
- index.html : Interface complète avec onglets Recherche / Sources / Favoris
  Filtres : marque, modèle dynamique, volant gauche/droite, année min/max, km max, budget max
  Fonctionnalités : fiche détaillée par voiture, sauvegarde favoris localStorage,
  export favoris, animation de chargement avec noms des sources
- app.js : Logique frontend + 20 voitures de démo en fallback
  Tente d'abord le backend /api/search, sinon utilise les données démo

BACKEND (backend/)
- server.js : Express + CORS + cron job toutes les 2h + Postgres
- routes/search.js : Route GET /api/search qui agrège toutes les sources en parallèle
- adapters/encarAdapter.js : Encar Korea (api.encar.com — PAS de clé requise)
- adapters/yahooAdapter.js : Yahoo Auctions JP (nécessite YAHOO_JP_APP_ID)
- adapters/beforwardAdapter.js : BE FORWARD (nécessite BEFORWARD_API_KEY)
- adapters/kcarAdapter.js : K Car scraper avec Playwright (pas de clé)
- jobs/refreshListings.js : Cron job qui pre-fetch et stocke en Postgres
- database/schema.sql : Tables listings, alerts, favorites

21 SOURCES CONFIGURÉES
🇯🇵 Enchères JP : USS, TAA, CAA, JU, HAA Kobe, Yahoo! Auctions JP
🇯🇵 Occasion JP : Goo-net Exchange, SBT Japan, BE FORWARD, CarSensor, Carused.jp, CardealPage, CarFromJapan
🇰🇷 Enchères KR : Autobell (Hyundai Glovis), Lotte Auto Auction, Heydealer, AJ Sellcar
🇰🇷 Occasion KR : Encar, K Car, Autowini, SK Encar

═══════════════════════════════════════════════════════════
  MISSION : CE QUE TU DOIS FAIRE
═══════════════════════════════════════════════════════════

1. INSTALLER LES DÉPENDANCES
   cd backend && npm install
   npx playwright install chromium

2. TESTER ENCAR EN PREMIER (pas de clé requise)
   - Lancer le backend : node src/server.js
   - Tester : curl "http://localhost:3001/api/search?q=Stinger&sources=encar"
   - Si ça retourne des vrais véhicules coréens → succès
   - Déboguer l'adapter si nécessaire (l'API Encar peut changer ses paramètres)

3. CONFIGURER RAILWAY
   - Initialiser Railway : railway init
   - Ajouter Postgres : railway add postgresql
   - Lancer le schema : railway run npm run db:init
   - Déployer : railway up
   - Récupérer l'URL publique et la mettre dans frontend/app.js

4. AJOUTER LES ADAPTERS MANQUANTS
   Les sources suivantes n'ont pas encore d'adapter — créer src/adapters/ pour :
   - goonetAdapter.js    → goo-net-exchange.com (API partenaire ou scraping)
   - sbtAdapter.js       → sbtjapan.com (scraping Playwright)
   - carsensorAdapter.js → carsensor.net (scraping Playwright)
   - autobellAdapter.js  → autobellglobal.com (API ou scraping)

5. SYSTÈME D'ALERTES EMAIL
   - Quand un utilisateur fait une recherche → option "créer une alerte"
   - Le cron job vérifie toutes les 2h si de nouvelles annonces matchent
   - Envoie un email via SendGrid (SENDGRID_API_KEY dans .env)
   - Créer : routes/alerts.js + le service email

6. AMÉLIORATION FRONTEND
   - Connecter le vrai backend (remplacer BACKEND_URL)
   - Afficher les vraies images des annonces (c.imageUrl)
   - Ajouter un lien direct vers l'annonce originale (c.listingUrl)
   - Afficher le délai restant pour les enchères (endsAt pour Yahoo JP)
